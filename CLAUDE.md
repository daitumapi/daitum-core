## Repository Structure

This is a monorepo housing three publishable Python libraries and a meta-package:

```
daitum-core/
├── daitum-model/             # daitum_model package (model + formula generation)
│   ├── src/daitum_model/     # source code
│   ├── tests/                # package tests
│   └── pyproject.toml
├── daitum-ui/                # daitum_ui package (UI definition generation)
│   ├── src/daitum_ui/        # source code
│   ├── tests/                # package tests
│   └── pyproject.toml
├── daitum-configuration/     # daitum_configuration package (model configuration)
│   ├── src/daitum_configuration/  # source code
│   ├── tests/                # package tests
│   └── pyproject.toml
├── pipelines/                # Shared scripts (lint.sh, etc.)
├── docs/                     # Unified Sphinx documentation (future)
└── pyproject.toml            # Root: daitum-core meta-package + shared tooling config
```

`daitum-core` is a meta-package with no source code — installing it installs all three
sub-packages.


## Daitum Platform Overview

These libraries are used to programmatically build optimisation models for the Daitum platform.
Each model consists of a model definition, model configuration, and UI definition. The final
output is a ZIP package of JSON files uploaded to the platform.

The libraries use fluent builder APIs:
- `daitum_model`: Define tables, fields, formulas, and formula functions
- `daitum_ui`: Define UI screens, navigation groups, components
- `daitum_configuration`: Build model configuration (algorithms, objectives, data sources)

All Daitum objects inherit from a `Buildable` base class and expose `.build()` for
JSON-compatible dict serialisation; the canonical output entry points are
`ModelBuilder.write_to_file()`, `UiBuilder.write_to_file()`, and
`ConfigurationBuilder.write_to_file()`.

### Import conventions

```python
import daitum_model
import daitum_model.formulas   # formula functions live inside daitum_model
import daitum_ui
import daitum_configuration
```


## Development Commands

### Virtual Environment

**ALWAYS use the virtual environment located at `venv/` in the project root.**

```bash
source venv/bin/activate
```

Install all packages in editable mode:

```bash
pip install -e daitum-model -e daitum-ui -e daitum-configuration
```

### Linting and Type Checking

**CRITICAL: When fixing linting issues, ALWAYS run the automated tools (Black, Ruff) instead of
manually editing code formatting.**

```bash
# Run all checks (Black, Ruff, MyPy) — same as CI pipeline
./pipelines/lint.sh

# Individual tools (always use venv)
source venv/bin/activate && black daitum-model/src daitum-ui/src daitum-configuration/src
source venv/bin/activate && ruff check daitum-model/src daitum-ui/src daitum-configuration/src
source venv/bin/activate && mypy --explicit-package-bases daitum-model/src daitum-ui/src daitum-configuration/src
```

**Workflow for fixing linting issues:**
1. Run Black first to auto-format.
2. Run Ruff to check/fix linting: `ruff check --fix`.
3. Add `# noqa: RULE_CODE` comments only when necessary for legitimate exceptions.
4. Never manually edit code formatting — let Black handle it.

### Running Tests

```bash
source venv/bin/activate && pytest
```
